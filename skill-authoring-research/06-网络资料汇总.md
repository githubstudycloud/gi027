## A. 官方一手资料

### A1. Anthropic 官方仓库 [`anthropics/skills`](https://github.com/anthropics/skills)
- **125k stars · 14.6k forks**（截至本次研究）
- 包含官方推荐目录：
  - `skills/` - Creative & Design / Development & Technical / Enterprise & Communication / Document 四大类示例
  - `spec/` - **Agent Skills 规范文档**
  - `template/` - 最小可复制模板
- 核心结论：`SKILL.md` **只需 `name` + `description` 两个字段**，其他都可选

### A2. Anthropic 工程博客（核心读物）
**[Equipping agents for the real world with Agent Skills](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)**

作者：Barry Zhang、Keith Lazuka、Mahesh Murag（2025-10-16 发布；2025-12-18 升级为开放标准）

#### 三层渐进披露（Progressive Disclosure）权威定义
| 层级 | 加载内容 | 触发条件 |
|------|---------|---------|
| **Level 1: Discovery** | metadata（name + description） | 启动时全量加载到 system prompt |
| **Level 2: SKILL.md body** | 完整 SKILL.md | 任务匹配描述时通过 Bash 读取 |
| **Level 3+: Bundled files** | references、scripts | SKILL.md 中显式引用时按需加载 |

#### 官方四大开发原则（直译）
1. **Start with evaluation**：先用真实任务跑 agent，找到能力缺口 → 增量补 Skill
2. **Structure for scale**：SKILL.md 过大时拆文件；互斥/罕用场景分路径；明确"代码是工具还是文档"
3. **Think from Claude's perspective**：观察 Claude 实际如何使用，迭代 `name` 和 `description`
4. **Iterate with Claude**：让 Claude 自我反思 → 把成功路径和失败教训写回 Skill

#### 安全告警（必读）
> Skills 通过指令和代码扩展能力，**恶意 Skill 可能引入漏洞或导致数据外泄**。
> - 仅安装可信来源的 Skill
> - 第三方 Skill 必须先审计：读所有文件、注意代码依赖、警惕外部网络访问指令

### A3. Agent Skills 开放标准 [agentskills.io](https://agentskills.io/)

**关键事实**：Anthropic 已于 2025-12-18 把 Skill 格式开源为跨平台标准。

#### 已支持 Skill 的客户端（部分）
Claude Code、Claude.ai、Claude API、Cursor、Kiro、Qodo、OpenCode、Goose、TRAE、VT Code、Letta、Snowflake Cortex Code、fast-agent、Google AI Edge Gallery、Piebald、Mux 等 **20+ 工具**。

→ **核心启示**：现在写一份 SKILL.md，可在多个 AI 客户端复用。

#### 标准目录骨架（官方）
```
my-skill/
├── SKILL.md          # 必需：metadata + 指令
├── scripts/          # 可选：可执行代码
├── references/       # 可选：文档
├── assets/           # 可选：模板、资源
└── ...               # 任意其他文件/目录
```

#### 三阶段定义（agentskills.io 原文）
1. **Discovery** - 仅加载 name + description
2. **Activation** - 任务匹配描述时读完整 SKILL.md
3. **Execution** - 跟随指令，按需执行脚本/加载引用文件

## B. 社区/独立专家观点

### B1. Simon Willison 博客（10/16 第一时间深度解读）
**[Claude Skills are awesome, maybe a bigger deal than MCP](https://simonwillison.net/2025/Oct/16/claude-skills/)**

#### 核心观点
- Skills **比 MCP 更值得期待**：MCP 是规范（hosts/clients/servers/三种 transport...），Skill 只是 Markdown + YAML
- **Token 效率**：每个 Skill 启动只占几十个 token，按需才完整加载；GitHub 官方 MCP 单独就吃几万 token
- **跨模型可用**：Skills 不是 Claude 专属，把目录指向 Codex CLI / Gemini CLI 也能用："读 pdf/SKILL.md 然后给我创建一个 PDF"
- **Skill 依赖代码执行环境**：Skills 只对带文件系统 + 命令执行能力的 agent 有意义
- Claude Code 的"create files"功能（PDF/DOCX/PPTX/XLSX）**实际就是用 Skills 实现的**
- "Cambrian explosion"预言：将出现远超 MCP 浪潮的 Skill 爆发

#### 实战示例：slack-gif-creator
- 描述：`Toolkit for creating animated GIFs optimized for Slack, with validators...`
- 关键设计：内置 `check_slack_size()` 校验函数，让 Claude 自动检查并重试
- 核心模式：**Skill 提供"工具 + 验证函数"**，让 Claude 闭环

## D. 提炼出的"高频成功模式"

汇总以上仓库后，发现优秀 Skill 的共同特征：

### 模式 1：Skill 内嵌"自校验函数"
- 案例：slack-gif-creator 的 `check_slack_size()`
- 价值：让 Claude 自己检查产物是否合规并重试
- **建议**：但凡有"输出约束"（大小、格式、范围），写一个验证脚本放进 `scripts/`

### 模式 2：核心 SKILL.md 极薄 + 厚 references/
- 案例：`document-skills/pdf/` 把 forms.md 拆出去
- 价值：填表场景才加载 forms.md，节省 token
- **建议**：当 Skill 里有"互斥子场景"，按子场景拆 reference 文件

### 模式 3：跨平台（多客户端）兼容声明
- 案例：`agent-skill-creator` 显式列出 14 个兼容客户端
- 价值：减少用户迁移成本
- **建议**：在 README 顶部写明"已在 X、Y、Z 测试"

### 模式 4：自学习闭环
- 案例：`jzOcb/writing-style-skill` 让 AI 写完后，从用户编辑差异自动提取规则更新 SKILL.md
- 价值：Skill 越用越准
- **建议**：高频使用的 Skill 加一个 `.skill-learning.log` 收集反馈

### 模式 5：Linter 工具
- 案例：`agent-sh/agnix` 提供 SKILL.md 静态检查
- 价值：发现 YAML 静默失败问题
- **建议**：批量管理 Skill 时引入 agnix 或类似工具

## G. 与本研究既有结论的交叉验证

| 本研究既有结论 | 网络资料验证 | 状态 |
|---------------|------------|------|
| description 是发现入口 | Anthropic 博客："Claude will use these when deciding whether to trigger" | ✅ 一致 |
| 渐进加载 3 层 | agentskills.io 官方定义 Discovery/Activation/Execution | ✅ 完全一致 |
| SKILL.md 控制在 < 500 行 | 官方建议"split content into separate files" | ✅ 一致（官方未给具体行数） |
| 只需 name + description | 官方原文："The frontmatter requires only two fields" | ✅ 一致 |
| 资源引用一层深 | 官方未明确，但 PDF skill 实例确实是单层 | ✅ 实践一致 |
| 跨平台可用 | agentskills.io 列出 20+ 客户端 | ✅ 强化 |

**新增结论（来自网络资料）**：
- 加上"自校验脚本"模式（slack-gif-creator 启发）
- 引入 Linter 工具（agnix）做提交前检查
- Skill 安全审计清单（来自 Anthropic 安全章节）
