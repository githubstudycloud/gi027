# Q&A Manager - 快速入门指南

## 🎯 简介

Q&A Manager 是一个强大的知识管理工具，帮助你在使用 Claude Code 时保存、组织和检索问答。当你问了很多问题，得到了很多答案时，这个工具能帮你有效管理这些知识。

## ✨ 核心特性

- **🔄 自动激活**: 当你想保存或搜索问答时自动启用，无需手动调用
- **🧠 智能存储**: 使用 MCP Memory 实现持久化和语义搜索
- **🏷️ 自动标签**: 自动提取技术栈标签，方便分类
- **📝 版本控制**: 支持更新答案，保留历史版本
- **🔍 强大搜索**: 语义搜索、标签过滤、时间筛选
- **📊 可视化**: 统计分析、知识图谱、学习路径

## 🚀 5分钟快速开始

### 步骤 1: 验证安装

检查 skill 是否已安装：

```bash
# 列出所有可用的 skills
ls .claude/skills/
```

你应该能看到 `qa-manager` 目录。

### 步骤 2: 保存你的第一个问答

在 Claude Code 中直接说：

```
保存这个问答：
问题：如何在 Git 中撤销最后一次提交？
答案：使用 git reset HEAD~1 保留更改，或 git reset --hard HEAD~1 丢弃更改
```

系统会自动：
1. 激活 qa-manager skill
2. 创建问题实体
3. 提取标签（#Git #版本控制）
4. 保存到知识库
5. 返回确认消息

### 步骤 3: 搜索问答

当你需要查找之前保存的问答时：

```
搜索关于 Git 的问答
```

或者：

```
找一下之前问过的撤销提交的方法
```

### 步骤 4: 查看统计

```
显示我的问答统计
```

恭喜！你已经掌握了基础用法 🎉

## ⚙️ 高级配置

### 启用自动化 Hooks（可选）

如果你想要更自动化的体验，可以配置 hooks：

1. **编辑配置文件**: `.claude/settings.local.json`

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node .claude/hooks/qa-manager/session-start.js",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

2. **效果**:
   - 每次启动会话时显示问答统计
   - 提示最近添加的问答
   - 显示快速操作指南

详细配置见：`.claude/hooks/qa-manager/README.md`

### 自定义标签规则

你可以定义自己的标签体系，例如：

```
技术栈标签: #React #Python #Docker
类别标签: #API #数据库 #性能
场景标签: #错误处理 #最佳实践 #故障排查
优先级标签: #常用 #重要 #待验证
难度标签: #入门 #中级 #高级
```

### 导出配置

支持多种导出格式：

- **Markdown**: 标准格式，易于阅读
- **Wiki**: 带导航和索引
- **JSON**: 结构化数据，便于程序处理
- **Obsidian**: 支持双向链接的知识库格式

## 💡 最佳实践

### ✅ DO - 推荐做法

1. **及时保存**: 得到有用答案后立即保存，不要拖延
2. **使用标签**: 为每个问答添加 2-5 个相关标签
3. **定期审查**: 每周查看新保存的问答，补充完善
4. **建立关联**: 链接相关问答，构建知识网络
5. **添加示例**: 尽量包含代码示例和实际用例
6. **记录上下文**: 说明问题出现的场景和环境
7. **标记状态**: 及时标记过时或需要更新的答案
8. **分类管理**: 使用清晰的分类体系

### ❌ DON'T - 避免做法

1. **不要过度保存**: 不是所有对话都需要保存
2. **避免重复**: 保存前搜索一下是否已有相似问答
3. **不要过于简略**: 答案要包含足够的细节和上下文
4. **避免依赖搜索**: 定期浏览和复习，不只是搜索
5. **不要忽视更新**: 发现错误或过时信息要及时更新
6. **避免杂乱标签**: 保持标签体系的一致性

## 🔧 故障排查

### Q: Skill 没有自动激活？

**检查清单:**
- ✅ 确认 `.claude/skills/qa-manager/SKILL.md` 存在
- ✅ 使用明确的触发词（如 "保存问答"、"搜索问答"）
- ✅ 检查 skill 描述是否清晰

**解决方法:**
```
# 手动激活 skill（不推荐，应该自动激活）
使用 qa-manager skill 保存这个问答...
```

### Q: Memory MCP 连接失败？

**检查清单:**
- ✅ 确认 Memory MCP server 已在 settings 中配置
- ✅ 检查 MCP server 是否正常运行
- ✅ 查看错误日志

**临时方案:**
使用文件系统备份（见下一节）

### Q: 搜索不到之前保存的问答？

**可能原因:**
1. 搜索关键词不够精确
2. 标签不匹配
3. 问答名称和内容差异较大

**解决方法:**
```
# 使用更宽泛的搜索
搜索所有问答

# 按标签浏览
显示标签为 #React 的所有问答

# 按时间浏览
显示最近保存的 20 个问答
```

### Q: 如何备份问答数据？

```
# 导出所有问答
导出所有问答为 markdown，保存到 qa-backup 目录

# 定期备份（推荐）
每周导出一次，上传到云盘或 Git
```
