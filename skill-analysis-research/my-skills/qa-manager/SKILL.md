# Q&A 知识管理器

## 概述
这个 skill 帮助你构建和维护个人问答知识库，支持智能保存、语义搜索、分类管理和持续改进。

## 核心功能

### 1. 💾 保存问答 (Save Q&A)
将问题和答案保存到知识图谱中，支持：
- 问题实体化：每个问题作为独立实体
- 答案关联：答案作为观察记录附加到问题上
- 自动标记：提取关键词和分类标签
- 上下文保存：记录时间、来源、相关代码等

**触发词示例：**
- "保存这个问答"
- "记录下来这个答案"
- "添加到知识库"
- "save this Q&A"

**保存格式：**
```
实体类型: Question
实体名称: 问题的简短描述（如 "如何优化React性能"）
观察记录:
  - 完整问题: [用户的完整问题]
  - 答案: [Claude 的答案]
  - 标签: #React #性能优化 #前端
  - 日期: 2025-12-21
  - 状态: verified/draft/outdated
  - 代码示例: [相关代码片段]
  - 相关链接: [参考资料]
```

### 2. 🔍 搜索问答 (Search Q&A)
通过语义搜索快速找到相关问答：
- 关键词搜索：根据问题内容搜索
- 标签过滤：按技术栈、类别筛选
- 模糊匹配：找到相似问题

**触发词示例：**
- "搜索关于 React 的问答"
- "找一下之前问过的数据库问题"
- "查看性能优化相关的答案"
- "search previous Q&A about..."

### 3. ✏️ 编辑问答 (Edit Q&A)
更新和改进已保存的问答：
- 更新答案：添加新的见解或更好的解决方案
- 补充说明：添加代码示例、参考链接
- 标记状态：标记为过时、已验证等
- 关联问题：链接相关的其他问答

**触发词示例：**
- "更新之前保存的答案"
- "修改这个问答"
- "给这个答案添加代码示例"
- "update the Q&A about..."

### 4. 📊 组织管理 (Organize)
通过关系图谱管理知识结构：
- 创建分类：前端、后端、DevOps、故障排查等
- 建立关联：链接相关问答
- 查看统计：查看问答数量、最近添加等
- 导出备份：导出为 markdown 文件

**触发词示例：**
- "显示所有问答分类"
- "查看问答统计"
- "列出最近的问答"
- "show my Q&A database stats"

## 数据结构设计

### 问答实体 (Question Entity)
```yaml
实体类型: Question
实体名称: 简短描述性标题
观察记录:
  - question: 完整的问题内容
  - answer: 完整的答案内容
  - tags: 标签列表（如 #React #API #性能）
  - timestamp: 创建/更新时间
  - status: verified/draft/outdated/deprecated
  - code_examples: 代码示例
  - references: 相关链接
  - follow_up: 后续问题列表
```

### 分类实体 (Category Entity)
```yaml
实体类型: Category
实体名称: 分类名称（如 "前端开发"）
关系: Category -> contains -> Question
```

### 标签实体 (Tag Entity)
```yaml
实体类型: Tag
实体名称: 标签名称（如 "React"）
关系: Question -> tagged_with -> Tag
```

## 工作流程

### 保存问答流程
1. 用户表达保存意图
2. Skill 自动激活
3. 提取问题和答案内容
4. 生成简短的实体名称
5. 自动提取标签和分类
6. 使用 `create_entities` 创建问题实体
7. 使用 `add_observations` 添加详细内容
8. 创建与分类/标签的关系
9. 确认保存成功

### 搜索问答流程
1. 用户输入搜索查询
2. Skill 自动激活
3. 使用 `search_nodes` 进行语义搜索
4. 解析搜索结果
5. 格式化显示匹配的问答
6. 提供查看详情或编辑的选项

### 更新问答流程
1. 用户指定要更新的问答
2. Skill 自动激活
3. 使用 `open_nodes` 获取现有问答
4. 使用 `add_observations` 添加新内容
5. 或使用 `delete_observations` 删除过时内容
6. 更新时间戳和状态
7. 确认更新成功

## 最佳实践

### 问题命名规范
- ✅ 使用简洁描述：`React性能优化-列表渲染`
- ✅ 包含关键技术：`MySQL索引优化策略`
- ❌ 避免过长：不要用完整问题句子作为名称
- ❌ 避免重复：检查是否已有类似问题

### 标签使用建议
- 技术栈标签：`#React` `#Python` `#Docker`
- 类别标签：`#API` `#数据库` `#性能`
- 场景标签：`#错误处理` `#最佳实践` `#故障排查`
- 优先级标签：`#常用` `#重要` `#待验证`

### 答案质量维护
- 定期审查过时答案
- 标记已验证的答案
- 添加实际使用的代码示例
- 记录解决问题的上下文
- 链接相关的其他问答

## 常见问题

**Q: 问答会保存多久？**
A: MCP Memory 持久化存储，除非手动删除否则永久保存。

**Q: 可以搜索答案内容吗？**
A: 可以！search_nodes 支持搜索观察记录的内容。

**Q: 如何避免重复保存？**
A: 保存前会自动搜索相似问题，提示你是否要合并或创建新问答。

**Q: 可以分享给团队吗？**
A: 可以导出为 markdown 分享，或使用共享的 MCP Memory 服务器。

**Q: 如何组织大量问答？**
A: 使用分类实体和标签系统，建立清晰的知识结构。

## 技巧提示

💡 **技巧 1:** 保存时添加具体的使用场景，方便以后搜索
💡 **技巧 2:** 用标签而非分类来组织，更灵活
💡 **技巧 3:** 定期回顾 "draft" 状态的问答，补充完善
💡 **技巧 4:** 为重要问答添加代码示例和测试结果
💡 **技巧 5:** 使用 "follow_up" 记录衍生的新问题
