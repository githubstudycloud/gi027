# 飞书操作助手 (Feishu Helper)

## 概述
这个 skill 帮助你快速使用飞书（Lark/Feishu）的各种功能，无需记忆复杂的 API 调用。

## 核心功能

### 1. 📨 消息管理

#### 发送消息
**触发词**: "发送飞书消息"、"给XX发消息"、"通知XX"

**功能**:
- 发送文本消息到个人或群组
- 发送富文本消息（支持 Markdown）
- 发送卡片消息（互动卡片）
- 批量发送消息

**使用示例**:
```
# 发送简单文本
"给张三发飞书消息：会议推迟到下午3点"

# 发送到群组
"在开发群发消息：代码已部署，请测试"

# 发送卡片
"发送通知卡片到项目组：新版本已发布"
```

**底层工具**: `mcp__mcp-feishu__im_v1_message_create`

#### 查看消息历史
**触发词**: "查看飞书消息"、"获取聊天记录"

**功能**:
- 获取群组或个人聊天历史
- 搜索特定内容的消息
- 查看最近的消息

**使用示例**:
```
"查看技术群最近的消息"
"搜索飞书中关于'上线'的消息"
```

**底层工具**: `mcp__mcp-feishu__im_v1_message_list`

### 2. 👥 群组管理

#### 创建群组
**触发词**: "创建飞书群"、"建立讨论组"

**功能**:
- 创建新群组
- 设置群名称、描述
- 添加群成员
- 配置群权限

**使用示例**:
```
"创建一个飞书群：项目组讨论，成员包括张三、李四"
"建立一个技术交流群"
```

**底层工具**: `mcp__mcp-feishu__im_v1_chat_create`

#### 查询群组
**触发词**: "查看我的飞书群"、"列出群组"

**功能**:
- 列出用户所在的所有群组
- 查看群组详情
- 获取群成员列表

**使用示例**:
```
"列出我所在的所有飞书群"
"查看项目组群的成员"
"显示开发群的详细信息"
```

**底层工具**:
- `mcp__mcp-feishu__im_v1_chat_list`
- `mcp__mcp-feishu__im_v1_chatMembers_get`

### 3. 📄 文档操作

#### 创建和编辑文档
**触发词**: "创建飞书文档"、"新建文档"、"导入文档"

**功能**:
- 创建新文档
- 导入 Markdown 到飞书文档
- 获取文档内容
- 编辑文档

**使用示例**:

#### 搜索文档

**功能**:
- 按关键词搜索文档

- 按创建者筛选

**使用示例**:
```

```

## 配置说明

### 必需配置

在 `~/.claude/settings.json` 或项目的 `.claude/settings.json` 中配置：

```json
{
  "mcp": {
    "feishu": {
      "appId": "cli_xxxxxxxxxxxx",
      "appSecret": "xxxxxxxxxxxxxxxx",
      "endpoint": "https://open.feishu.cn"
    }
  }
}
```

```bash
export FEISHU_APP_ID="cli_xxxxxxxxxxxx"
export FEISHU_APP_SECRET="xxxxxxxxxxxxxxxx"
```

## 智能识别

Skill 会智能识别以下场景：

1. **检测飞书相关操作**: 任何提到"飞书"、"Feishu"、"Lark" 的请求
2. **检测具体功能**: 消息、文档、Base、群组等关键词
3. **检测操作类型**: 创建、查询、更新、删除等动作
4. **自动选择工具**: 根据用户意图选择合适的 MCP 工具

## 错误处理

当遇到错误时，Skill 会：

1. **权限错误**: 提示需要相应的飞书权限
2. **参数错误**: 引导用户提供必要的参数
3. **API 错误**: 显示友好的错误信息
4. **网络错误**: 提示检查网络连接

## 最佳实践

1. **使用明确的标识**: 使用文档名、表格名、群组名而不是 ID
2. **批量操作**: 一次性完成相关操作，减少 API 调用
3. **权限管理**: 操作前确保有足够的权限
4. **数据备份**: 重要操作前先查询确认
5. **测试环境**: 在测试环境先验证复杂操作

## 限制说明

1. **API 限额**: 飞书 API 有调用频率限制
2. **权限范围**: 需要配置相应的应用权限
3. **数据大小**: 大批量数据需要分页处理
4. **文件上传**: 大文件需要分片上传

## 常见问题

**Q: 如何获取应用凭证？**
A: 在飞书开放平台创建应用，获取 App ID 和 App Secret

**Q: 如何获取用户 ID？**
A: 使用"查找飞书用户"功能，通过邮箱或手机号查询

**Q: 如何获取群组 ID？**
A: 使用"查看我的飞书群"功能，会列出所有群组及其 ID

**Q: 消息发送失败？**
A: 检查：1) 应用权限 2) 用户/群组 ID 正确 3) 消息格式正确

**Q: Base 操作失败？**
A: 确保：1) Base 存在 2) 有操作权限 3) 字段类型匹配
