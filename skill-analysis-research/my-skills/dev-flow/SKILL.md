## 根据参数给出精准建议

如果 `$ARGUMENTS` 指定了阶段：

- **`planning` / `需求`** → 建议先用 `/requirements`，再 `/sprint-plan`
- **`coding` / `开发`** → 建议用 `/new-feature`（一站式）
- **`testing` / `测试`** → 建议用 `/gen-tests` 补充测试
- **`review` / `审查`** → 建议先 `/check-standards` 自检再 `/review-pr`
- **`release` / `发布`** → 使用 `/release`
- **`bug` / `故障`** → 生产用 `/incident`，开发中用 `/bug-fix`
- **`new project` / `新项目`** → 先 `/onboard`（了解项目）

## 常见问题路由

| 我想要... | 使用 |
|-----------|------|
| 做一个全新功能 | `/new-feature 功能描述` |
| 理解一个陌生项目 | `/onboard` |
| 写一个新接口 | `/api-spec 资源名` |
| 建一张新表 | `/db-schema 实体名` |
| 写设计文档 | `/design 功能名` |
| 修复一个 Bug | `/bug-fix 错误信息` |
| PR 合并前检查 | `/check-standards` |
| 给别人 Review PR | `/review-pr 123` |
| 打版本发布 | `/release minor` |
| 生产挂了 | `/incident 故障描述` |
| 看看有多少技术债 | `/tech-debt` |
| 需求拆成任务 | `/sprint-plan docs/prd.md` |
