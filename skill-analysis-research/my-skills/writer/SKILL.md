# 智能写作审核编排器

对 `$ARGUMENTS` 中的文稿进行智能分类并驱动专家审核。

---

## 第一步：识别写作类型

分析输入内容，识别属于以下哪种类型：

| 代码 | 类型 | 特征关键词 |
|------|------|-----------|
| `tech` | 技术文档 | API、代码、架构、接口、部署、README |
| `business` | 商业文案 | 报告、提案、方案、合同、邮件、汇报 |
| `creative` | 创意写作 | 故事、小说、剧本、散文、诗歌、角色 |
| `academic` | 学术写作 | 论文、研究、文献、假设、实验、摘要 |
| `marketing` | 营销文案 | 广告、促销、卖点、用户痛点、转化、落地页 |
| `news` | 新闻稿 | 公告、发布、新闻、媒体、事件、报道 |
| `social` | 社交媒体 | 微博、小红书、朋友圈、话题、互动、粉丝 |

如果用户**已明确指定类型**，跳过识别，直接使用指定类型。

如果**无法确定**，输出：
> "我识别到这可能是 [类型A] 或 [类型B]，请问是哪种类型？"

---

## 第二步：确认审核范围

向用户简要确认：

```
识别类型：[类型名称]
审核维度：结构 / 语言 / 受众适配 / 专业度 / 可读性
预计输出：评分报告 + 逐段改进建议

是否继续？(直接回复"继续"或提出调整)
```

---

## 第三步：调度子专家审核

根据识别的类型，使用 Agent 工具调用对应子专家：

| 类型 | 调用的 Agent |
|------|------------|
| `tech` | `writer-tech-expert` |
| `business` | `writer-business-expert` |
| `creative` | `writer-creative-expert` |
| `academic` | `writer-academic-expert` |
| `marketing` | `writer-marketing-expert` |
| `news` | `writer-news-expert` |
| `social` | `writer-social-expert` |

将完整文稿和审核要求传递给子专家 Agent。

---

## 第四步：整合输出审核报告

汇总子专家的审核结果，按以下结构输出：

```markdown
# 写作审核报告

**文稿类型**: [类型]
**审核时间**: [时间]
**文稿字数**: [字数]

---

## 综合评分

| 维度 | 得分 | 等级 |
|------|------|------|
| 结构  | xx/20 | ⭐⭐⭐ |
| 语言  | xx/20 | ⭐⭐⭐ |
| 受众  | xx/20 | ⭐⭐⭐ |
| 专业度 | xx/20 | ⭐⭐⭐ |
| 可读性 | xx/20 | ⭐⭐⭐ |
| **总分** | **xx/100** | **[等级]** |

---

## 核心发现

### 优点
- ...

### 待改进
- ...

---

## 逐段建议

[按原文段落顺序给出具体改进意见]

---

## 改写示例

[选取最需改进的 1-2 段给出完整改写版本]
```

---

## 注意事项

- 保持中立客观，不主观评价写作主题好坏
- 改写建议须保持原文核心意图不变
- 专业术语不得擅自修改
- 评分须有具体依据，不得空泛给分
