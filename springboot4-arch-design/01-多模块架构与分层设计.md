## 2. 模块分类与依赖方向

```
[业务项目] ──depends on──► [enterprise-starter-*]
                                │
                                ▼
                         [enterprise-core / data / mq / security / ai / config]
                                │
                                ▼
                         [enterprise-common]
                                │
                                ▼
                         [enterprise-bom]（仅版本）
```

**铁律**：
- 依赖只能**向下**，禁止反向；
- `*-api` 模块**不依赖任何实现**，只定义接口、DTO、异常；
- `*-impl` / `*-starter` 才依赖具体技术。

## 3. 业务项目内部分层（同一 deploy unit）

```
┌──────────────────────────────────────────┐
│ adapter  (in: web/mq-listener  out: rpc/db) │  ← 与外界唯一接触面
├──────────────────────────────────────────┤
│ application (use case 编排，事务边界)        │  ← @PartialFailureTolerant 标注处
├──────────────────────────────────────────┤
│ domain (实体、聚合、领域服务、Port 接口)      │  ← 不依赖任何框架
├──────────────────────────────────────────┤
│ infrastructure (Port 实现、DB、外部 client) │
└──────────────────────────────────────────┘
```

> 采用**六边形（Ports & Adapters）**而非传统三层；好处：内层无 Spring 注解，可纯 JUnit 测试，且替换基础设施时不动业务代码。

## 4. 模块通信规范（关键回应你的问题）

### 4.1 跨子项目（跨进程）

- 强制走 `*-client` 模块；
- 客户端封装：Feign / RestClient + Resilience4j（熔断 / 重试 / 超时 / 隔离）+ TraceId 透传；
- 接口契约用 OpenAPI yaml 双向校验，CI 阶段比对。

### 4.2 同一子项目内（同进程）

业务上有"想解耦"的诉求，但**不允许**伪 HTTP 调用。统一做法：

| 场景 | 方案 |
|------|------|
| 同层之间想解耦 | 定义 `Port` 接口（domain 层），实现放 infrastructure；通过 Spring 注入 |
| 想做"插件式"扩展 | 用 `SPI` 模式：`@Component` + 自定义 `@ExtensionPoint`，由 `ExtensionManager` 路由 |
| 想做"可替换实现" | 用 `@ConditionalOnProperty` / `@ConditionalOnMissingBean` |
| 异步解耦 | 用 `ApplicationEventPublisher`（同 JVM）或 MQ（跨 JVM） |

**禁止**：同 JVM 内为了"解耦"而引入 HTTP 自调用（性能差、链路噪声、难定位）。

### 4.3 模块间共享 DTO

- 仅允许通过 `*-api` 模块共享；
- DTO 用 Java 21 `record`，不可变；
- 跨进程版本演进遵循"加字段不删字段、删字段经过 2 个版本 deprecated"。

## 5. 业务项目交付形态

| 形态 | 适用 | 引入方式 |
|------|------|---------|
| **单体业务项目** | 中小型业务、内部系统 | `enterprise-starter-monolith` |
| **微服务业务项目** | 大型业务、独立团队 | `enterprise-starter-microservice` |
| **任务型项目（Job / Worker）** | 异步消费、定时任务 | `enterprise-starter-worker` |
| **BFF / 网关插件** | 对外/对前端 | `enterprise-starter-gateway-plugin` |

> 通过 `archetype` 一键生成，强制满足分层、规范、CI 配置。

## 6. 自我修订记录

| 轮次 | 关注点 | 修订点 |
|------|--------|--------|
| R1 | 多项目可拆性 | 拆出 `*-api` / `*-client` / `*-starter` 三件套，避免业务项目被迫引入实现 |
| R2 | 同 JVM 解耦 | 明确禁止 HTTP 自调用，引入六边形 Port 模型 |
| R3 | 部分失败 | 在 application 层独立标记 `@PartialFailureTolerant`，与默认事务边界区分 |
