# 05 - 消息中间件与异步流程设计

> 版本：v1.0 | 日期：2026-04-27 | 状态：草稿

---

## 1. MQ 选型矩阵

| MQ | 定位 | 适用场景 | 是否纳入 |
|----|------|---------|---------|
| **RocketMQ 5.x** | 业务消息主选 | 事务消息、定时、顺序、广播 | ✅ 默认 |
| **Kafka 3.x** | 高吞吐流式 | 数据管道、日志、CDC、AI 数据流 | ✅ 流式场景 |
| **RabbitMQ** | 复杂路由 | 与老系统集成 | ⚠️ 仅兼容 |
| **Pulsar** | 多租户 / 分层存储 | 超大规模 | ❌ 暂不引入 |
| **Redis Stream** | 轻量异步 | 单 JVM 替代场景 | ⚠️ 简单异步 |

## 2. 统一抽象（关键）

```
mq-api/
├── MessageProducer          // send / sendAsync / sendTx / sendDelay / sendOrdered
├── MessageConsumer          // @MessageListener
├── Message<T>               // header + payload + traceId + tenantId
├── MessageInterceptor       // 链：日志、加密、脱敏、追踪、幂等
└── DeadLetterHandler

mq-rocketmq / mq-kafka / mq-redisstream  ← 各自实现
```

- 业务代码**只 import `mq-api`**，切换底层 MQ 不改业务；
- 各实现差异（事务消息、顺序消息）通过 capability 标志暴露：`producer.supports(TX)`。

## 3. 可靠性模式

### 3.1 生产端

| 模式 | 何时用 | 实现 |
|------|--------|------|
| 普通投递 | 容忍丢失（如埋点） | 异步 + 重试 |
| 至少一次 | 业务事件主线 | 同步发送 + 失败入 outbox |
| **事务消息（推荐）** | 跨库一致性 | RocketMQ 事务消息 / 自实现 outbox |
| 延迟消息 | 定时提醒、超时关单 | RocketMQ 原生 |
| 顺序消息 | 同实体的有序变更 | hash by entityId |

### 3.2 消费端

- **必须幂等**：消费者侧用 `core-idempotent` 模块，依据 `messageId + bizKey` 去重；
- **失败重试**：本地重试 3 次（指数退避）→ 进入死信 topic → 人工介入；
- **死信治理**：死信必须有归属人 + SLA + 监控告警。

### 3.3 部分失败（呼应你的关注点）

批量消息处理（如 1000 条订单批量退款）：

```java
@PartialFailureTolerant
public PartialResult<RefundResult> handle(List<RefundCmd> cmds) {
    // 子项独立事务；失败不影响其他项；统一收集
}
```

- 框架统一提供 `PartialResult<T>`：成功列表 + 失败列表（含原因 + 是否可重试）；
- 失败项落 `t_partial_failure` 表，由独立 worker 重试或人工补偿；
- 调用方拿到结果可决定整体策略（部分成功 = 接受 / 拒绝 / 补偿）。

## 4. 异步执行抽象（同 JVM）

| 工具 | 适用 |
|------|------|
| `ApplicationEventPublisher` | 同 JVM 解耦（事务后置事件 `@TransactionalEventListener`） |
| `@Async` + DynamicTp | 简单异步 |
| Project Reactor | 响应式流 |
| Virtual Threads（Java 21） | IO 密集场景，替代线程池 |

> 选择规则：**优先事件 → 其次虚拟线程 → 再次 @Async → 跨 JVM 才用 MQ**。

---

## 5. 链路追踪与可观测

- 所有消息 header 强制带 `traceId / spanId / tenantId / userId`；
- 跨 MQ 自动透传，与 Skywalking / OpenTelemetry 集成；
- 监控指标：发送 TPS、消费 TPS、堆积、消费延迟、重试次数、死信数；
- 告警：堆积 > 阈值、消费延迟 > SLA、死信增量。

---

## 6. 安全

- 敏感字段（手机号、身份证）发送前自动脱敏 / 加密（拦截器统一处理）；
- Topic 鉴权：生产 / 消费白名单（运维侧）；
- 跨租户消息禁止串扰：消费时强校验 `tenantId`。

| 轮次 | 修订点 |
|------|--------|
